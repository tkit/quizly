# 中学年以上向け学習Webサイト 要件定義書

## 1. プロジェクト概要
* **ターゲットユーザー**: 中学年以上（小学校高学年〜中学生を中心）
* **目的**: 楽しく継続して学習できる、問題演習形式のWebサイト
* **利用環境**: iPad（メイン）、スマートフォン（一部）
* **提供形式**: Webサイト（ブラウザからのアクセスを想定）
* **文言トーン方針**: `docs/persona.md` に準拠
* **認証・認可仕様の正本**: `docs/auth_spec.md`

## 2. 機能要件

### 学習者向け機能
1. **簡易ログイン機能**
   * 子どもはプロフィールを選択して学習開始（子PINは不要）。
   * 保護者が認証済みであることを前提に、学習開始のハードルを下げる。
   * 学習記録は選択した子プロフィールに紐づけて蓄積する。
2. **問題演習（テスト）機能**
   * ジャンル（例: 算数、社会など）を選択し、出題数（例: 5問、10問など）を指定して開始。
   * 問題はテキスト形式の「選択肢形式」で出題。
   * 1問解答するごとに、即座に正解・不正解の判定と「解説」を表示する。
3. **学習記録機能**
   * いつ、何のジャンルを、何問解いたか、それぞれの正解/不正解の結果を蓄積する。
   * データベースと連携し、iPadやスマートフォンなど複数端末で学習記録を共有・同期できるようにする。
4. **復習（再チャレンジ）機能**
   * 過去に「不正解」だった問題だけをピックアップして、再度テストを受けられる機能。
5. **将来的な拡張への考慮**
   * 問題への画像表示（図形問題や、歴史上の人物画像などへの対応）。

### 管理者（親）向け機能
1. **問題の追加・削除・更新**
   * 当面は管理画面のUIを作り込まず、IT系エンジニアの親が運用しやすい方法（ファイルベースの管理）を採る。
   * CSVやJSON、またはGitHub等で問題データを管理し、それをDBやアプリに反映させるシンプルな仕組みを想定。
2. **将来的な拡張への考慮**
   * 親がより簡単に問題を管理できるよう、UI上からのアップロード形式の管理機能を提供。

## 3. 非機能要件・技術スタック

* インフラやバックエンドは、ストレージ等を含めて**可能な限り無料**で運用できる構成とする。

| 用途 | 選定技術/サービス | 選定理由・補足 |
| :--- | :--- | :--- |
| **フロントエンド** | Next.js (React) | 開発体験が良く、Vercelとの親和性が高い。 |
| **UI・スタイリング** | Tailwind CSS + shadcn/ui | 美しく整ったUIを素早く構築でき、カスタマイズ性が高い。 |
| **ホスティング** | Vercel | 無料枠が充実しており、Next.jsの動作環境として最適。 |
| **データベース** | Supabase | 充実した無料枠。学習記録の蓄積、ユーザー管理（簡易認証）の基盤として利用。 |
